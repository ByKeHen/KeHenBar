</div>
<p>
    &nbsp; &nbsp; &nbsp; &nbsp; 触发器是一种操作数据库表数据的行为，他会在目标表执行增删改查等操作时触发。通常我们会用他来保证数据完整性，例如有一个帖子表，帖子评论表，帖子评论点赞表。那么当删除帖子后，对应的评论以及点赞信息也要清空，否则就会形成垃圾数据占用空间，那么这时候就可以选择触发器来完成这个工作。
</p>

</p>
<p>
    &nbsp; &nbsp; &nbsp; &nbsp; sql语句的写法如下：
</p>
<pre class="brush:c#;toolbar:false">update users set plcount=plcount+1 where id=[sys_userid]</pre>
<blockquote class="site-tips site-text">
    [sys_userid]是系统全局参数，模版页面的任何地方都可以使用，他表示当前登录用户的用户id。
</blockquote>
<p>
    &nbsp; &nbsp; &nbsp; &nbsp; 帖子被浏览时要更新帖子浏览次数，这个用触发器怎么表示？目标表选择帖子表，选中查询事件。sql语句如下，&nbsp;
</p>
<pre class="brush:c#;toolbar:false">update forums 
 set seecount=seecount+1 
where id=[forumsid]</pre>
<p>
    <br/>

<blockquote class="site-tips site-text">
    &nbsp; &nbsp; &nbsp; &nbsp; [forumsid]表示forums表的id字段的值，在这里这个标记会被替换成查询出来的帖子的id。至于为什么要这么写，为什么这么写会被解释成帖子id，很难理解。 &nbsp;<br/>&nbsp; &nbsp; &nbsp; &nbsp; 首先这个触发器是给帖子表也就是forums创建的，是在formns表执行单条语句查询操作后被执行的。也就是当用户打开帖子内容时被执行（查询类型的触发器都只有这种情况下被执行，查询多条数据时不会执行）。每种类型的触发器执行时都会把该条数据作为数据源传入触发器用来解析触发器中的参数，解析规则为参数用一对英文中括号包裹，以表名+字段名命名。&nbsp;<br/>&nbsp; &nbsp; &nbsp; &nbsp; 因此这个触发器写的参数[forumsid]会被解释成显示的这个帖子的id。
</blockquote>
